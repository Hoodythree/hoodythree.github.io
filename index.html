为Django项目创建数据库并建立模型

发表于 2019-09-15 更新于 2019-09-17

1. 数据库配置

  1. 打开MyProject中同名App下的settings.py文件.默认情况下,配置使用Django自带的SQLite数据库,配置如下: 

    DATABASES = {
  'default': {
      'ENGINE': 'django.db.backends.sqlite3',
      'NAME': os.path.join(BASE_DIR, 'db.sqlite3'),
  }
}

    如果你需要使用其他的数据库,则需要修改DATABASES中的以下字段: 

    ENGINE--可选有'django.db.backends.mysql'或'django.db.backends.oracle'等其它可用后端
NAME--数据库的名字

    如果你没有使用默认的SQLite数据库,则还需要配置USER PASSWORD和HOST

    此外,关注一下文件头部的INSTALLED_APPS选项,这里包括了你会在Project中使用的Django应用.应用可以被多个Project使用,你也可以打包他们供别人使用.

    通常， INSTALLED_APPS 默认包括了以下 Django 的自带应用：

    django.contrib.admin -- 管理员站点， 你很快就会使用它。
django.contrib.auth -- 认证授权系统。
django.contrib.contenttypes -- 内容类型框架。
django.contrib.sessions -- 会话框架。
django.contrib.messages -- 消息框架。
django.contrib.staticfiles -- 管理静态文件的框架。

    默认开启的某些应用需要至少一个数据表，所以，在使用他们之前需要在数据库中创建一些表。需要执行以下命令：

    py manage.py migrate

    这个migrate命令检查INSTALLED_APPS设置，为其中的每个应用创建需要的数据表，至于具体会创建什么，这取决于你的mysite/settings.py 设置文件和每个应用的数据库迁移文件

  2. 创建models
    MVC是Web开发中常用的一种设计模式.其中的M即是model,它以类的形式对数据库中的表进行映射,官网教程中给出了这样一个例子:

    from django.db import models

class Question(models.Model):
    question_text = models.CharField(max_length=200)
    pub_date = models.DateTimeField('date published')
class Choice(models.Model):
    question = models.ForeignKey(Question, on_delete=models.CASCADE)
    choice_text = models.CharField(max_length=200)
    votes = models.IntegerField(default=0)

    代码非常直白,对每个表的属性和表与表之间的关系进行了定义.

  3. 激活models
    接下来我们需要对定义的模型进行激活.
    首先,需要将应用加入到我们的项目之中,具体做法是在配置类INSTALLED_APPS中添加设置:

    'polls.apps.PollsConfig'

    然后,运行以下命令:

    ...\> py manage.py makemigrations polls

    通过运行 makemigrations 命令，Django 会检测你对模型文件的修改（在这种情况下，你已经取得了新的），并且把修改的部分储存为一次迁移.

    迁移是非常强大的功能，它能让你在开发过程中持续的改变数据库结构而不需要重新删除和创建表 - 它专注于使数据库平滑升级而不会丢失数据。我们会在后面的教程中更加深入的学习这部分内容，现在，你只需要记住，改变模型需要这三步:

    • 编辑 models.py 文件，改变模型
    • 运行 python manage.py makemigrations 为模型的改变生成迁移文件
    • 运行 python manage.py migrate 来应用数据库迁移

  4. 初试API

初识Django

发表于 2019-09-14 更新于 2019-09-17

1.Django是什么?

Django是基于python的Web Application Framework,引用官网的一段介绍:

Django is a high-level Python Web framework that encourages rapid development and clean, pragmatic design. Built by experienced developers, it takes care of much of the hassle of Web development, so you can focus on writing your app without needing to reinvent the wheel. It’s free and open source.

2.为什么使用Django?

对我而言,我需要一个快速开发框架,最好能包括尽量多的工具,Django完美满足这些特点.
与Flask相比,Django提供了大量的工具,比如:

  1. 表单–Django Form
  2. csrftoken登陆认证鉴权–Django 自带了 backend 和 auth 模块。
  3. ORM–Django ORM
  4. DBMigration–Django Migration
  5. 缓存–Django Cache…
    做到了开箱即用.

3.如何使用Django?

  • 在Windows上安装Django

    step1. 安装python和pip包管理工具
    step2. 安装virtualenv和virtualenvwrapper这两个工具

    virtualenv:用于创建一个独立的虚拟python环境,其中每个环境都有自己的库和site-package
vittualenvwrapper:简化virtualenv的使用

    step3. Install Django 

    pip install django

  • Pycharm搭建第一个Diango项目
    有两种方法创建一个Django应用,一是在创建Django项目的同时创建一个Django应用,二是通过在Terminal中输入以下命令:

    py manage.py startapp appname

    Django应用的目录结构如下:

    +yourApp/
    + migrations/
        __init__.py
    __init__.py
    admin.py
    apps.py
    models.py
    tests.py
    views.py

    其中各个文件的作用如下:

    • migration :用于连接网站和数据库
    • __initial.py:告诉python将这个app视作一个python应用
    • admin.py:每个网站都会有一个admin section,用于实现管理功能
    • apps.py:app的配置文件
    • models.py: 我的理解是model是数据库的一个映射,它指明了app中的数据是如何进入到数据库的
    • test.py:测试文件,用于debug
    • views.py:views本质上是python函数,接受用户的request并respond相应的结果

  • Hello world in Django
    首先,在yourApp/views下创建一个view

    from django.http import HttpResponse

def index(request):
return HttpResponse('<h1>Hello World</h1>')

    然后,在yourApp下创建一个urls.py文件,定义一个url pattern

    from django.urls import path
from . import views

urlpatterns = [
    path('', views.index, name='index')
]

    最后,将yourApp/urls.py中创建的url pattern加入到根目录下urls.py的path中

    from django.contrib import admin
from django.urls import path, include

urlpatterns = [
    path('admin/', admin.site.urls),
    path('helloworld/', include('HelloWorld.urls')
]

  • 运行Hello World
    在浏览器中输入http://127.0.0.1:8000/helloworld/
    Got it!
